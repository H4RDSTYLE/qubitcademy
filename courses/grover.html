01

El problema de búsqueda no estructurada

⏱ 18 min🌱 Principiante✨ Gratis

Imagina que tienes una caja con un millón de fichas, cada una con un número en el reverso. Solo una ficha tiene el número que buscas. La única forma de encontrarla es darle la vuelta una por una. En promedio, necesitarás mirar 500.000 fichas antes de encontrar la correcta. Lov Grover descubrió en 1996 que un ordenador cuántico puede hacer esto en solo ~1.000 pasos.

Búsqueda clásica: el límite inferior

El problema de búsqueda no estructurada es fundamental en informática: dada una base de datos de N elementos y una función f que devuelve 1 para el elemento buscado y 0 para el resto, ¿cuántas consultas a f necesitas para encontrar el elemento con certeza?

Clásicamente, en el peor caso necesitas N consultas y en el caso promedio N/2. No hay truco posible: si los elementos no tienen ninguna estructura (no están ordenados, no hay patrón), no hay forma de evitar ir uno por uno. Esto se puede demostrar formalmente: cualquier algoritmo clásico determinista o probabilístico necesita Ω(N) consultas.

Búsqueda no estructurada vs. estructurada

La búsqueda binaria (buscar en una lista ordenada) requiere O(log N) pasos clásicamente — pero solo porque hay estructura (el orden). En una base de datos no estructurada (sin orden, sin índice, sin patrón), no hay atajos clásicos posibles. Grover da una ventaja cuadrática incluso en ausencia de estructura: √N consultas para encontrar el elemento con alta probabilidad.

El resultado de Grover

En 1996, Lov Grover publicó un algoritmo cuántico que encuentra el elemento marcado en O(√N) consultas al oráculo. Para N = 1.000.000, eso pasa de 500.000 consultas promedio a solo ~785. Para N = 10¹²: de 500.000 millones a ~1.000.000. La ventaja crece sin límite.

Aún más impresionante: Grover demostró que ningún algoritmo cuántico puede hacerlo mejor. O(√N) es el límite inferior cuántico para búsqueda no estructurada. Es óptimo.

Clásico: O(N) consultas     Grover: O(√N) consultas     Ventaja: cuadrática

¿Por qué "cuadrática" y no "exponencial"?

A diferencia de Shor (que da ventaja exponencial para factorizar), Grover da ventaja cuadrática. Para N = 2¹²⁸ (el espacio de claves AES-128): √N = 2⁶⁴. Eso sigue siendo enorme, pero no hace AES roto — simplemente equivale a reducir la seguridad de 128 bits a 64 bits efectivos. Por eso AES-256 (que quedaría en 128 bits efectivos) sigue siendo considerado seguro en la era cuántica.

🏆

Óptimo provado: Bennett, Bernstein, Brassard y Vazirani demostraron en 1997 que cualquier algoritmo cuántico de búsqueda necesita al menos Ω(√N) consultas. Grover no solo es bueno — es el mejor posible. Es uno de los pocos resultados de complejidad cuántica donde conocemos tanto el límite superior (Grover) como el inferior (BBBV).

Superposición uniforme: el punto de partida

⏱ 20 min🌱 Principiante✨ Gratis

Antes de poder "buscar" en una base de datos cuántica, necesitamos que todos los posibles estados estén disponibles simultáneamente. El primer paso de Grover crea la superposición uniforme: todos los N estados con exactamente la misma amplitud. Es el equivalente cuántico de "mirar todos los cajones a la vez".

¿Qué es la superposición uniforme?

Con n qubits podemos representar 2ⁿ = N estados base: |0⟩, |1⟩, |2⟩, …, |N−1⟩. La superposición uniforme es el estado donde todos tienen exactamente la misma amplitud:

|s⟩ = (1/√N) Σₓ |x⟩ = (1/√N)(|0⟩ + |1⟩ + |2⟩ + … + |N−1⟩)

La amplitud de cada estado es 1/√N, y la probabilidad de medir cualquier estado es (1/√N)² = 1/N. Todos igualmente probables — máxima ignorancia, máxima superposición.

Cómo preparar la superposición uniforme

Crear |s⟩ desde el estado inicial |0⟩ⁿ es sorprendentemente simple: basta con aplicar una puerta Hadamard a cada qubit independientemente.

H⊗ⁿ |0⟩ⁿ = |s⟩

¿Por qué funciona? La puerta H transforma |0⟩ en (|0⟩+|1⟩)/√2. Aplicada a n qubits independientes, el producto tensorial genera todas las combinaciones posibles de 0s y 1s — exactamente todos los estados de n bits — con igual amplitud.

Para n = 3 qubits (N = 8):

(1/√8)(|000⟩+|001⟩+|010⟩+|011⟩+|100⟩+|101⟩+|110⟩+|111⟩)
El poder de la superposición

Con n = 300 qubits, la superposición uniforme representa simultáneamente 2³⁰⁰ estados — más que el número de átomos en el universo observable. Preparar este estado toma exactamente n puertas Hadamard y un tiempo constante. Ningún ordenador clásico puede siquiera almacenar todos esos estados, mucho menos procesarlos todos a la vez.

El estado |s⟩ como vector geométrico

Una forma muy útil de visualizar Grover es geométrica. Considera el espacio de 2 dimensiones formado por:

  • El vector |ω⟩: el estado marcado (la solución que buscamos)
  • El vector |ω⊥⟩: la superposición uniforme de todos los estados excepto |ω⟩

El estado inicial |s⟩ está casi alineado con |ω⊥⟩ (muy lejos de |ω⟩), con un ángulo θ donde sin(θ) = 1/√N. Grover rota este vector hacia |ω⟩ mediante una serie de reflexiones. Después de ~π√N/4 iteraciones, el vector apunta casi exactamente a |ω⟩ y la medición da el resultado correcto con alta probabilidad.

🎯

Solo n puertas para N estados: Preparar la superposición uniforme de N = 2ⁿ estados toma exactamente n puertas Hadamard — una operación de tiempo O(n) = O(log N). Un ordenador clásico necesitaría O(N) pasos solo para listar todos los estados. Aquí ya hay una ventaja exponencial en la preparación del estado inicial.

El oráculo cuántico

⏱ 22 min🟡 Intermedio✨ Gratis

El oráculo cuántico es el componente que sabe cuál es la respuesta correcta. No "busca" — simplemente reconoce la solución cuando la ve y la marca. En el modelo de complejidad cuántica, el oráculo es una caja negra y contamos cuántas veces la consultamos. La magia de Grover es que necesita consultarla muy pocas veces.

¿Qué hace el oráculo?

Formalmente, el oráculo es una función booleana f: {0,1}ⁿ → {0,1} que devuelve 1 para el elemento marcado |ω⟩ y 0 para todos los demás. En la versión cuántica, el oráculo actúa como una puerta unitaria U_f que:

U_f|x⟩ = (−1)^f(x)|x⟩

Si x ≠ ω: U_f|x⟩ = |x⟩ (sin cambio)
Si x = ω: U_f|ω⟩ = −|ω⟩ (invierte la fase)

El oráculo no mide ni colapsa el estado — simplemente cambia el signo de la amplitud del estado marcado. Este truco se llama phase kickback (retroceso de fase).

Phase kickback: el truco fundamental

Para implementar la inversión de fase sin medir, se usa un qubit auxiliar en el estado |−⟩ = (|0⟩−|1⟩)/√2. Cuando aplicas la función f controlada por |x⟩ sobre el qubit auxiliar: si f(x)=1, la fase −1 "rebota" al qubit de control. Si f(x)=0, no pasa nada. Este mecanismo permite que el oráculo marque el estado correcto con una fase sin destruir la superposición.

El efecto del oráculo sobre la superposición

Antes del oráculo, todos los estados tienen amplitud +1/√N. Después del oráculo, el estado marcado |ω⟩ tiene amplitud −1/√N y todos los demás siguen con +1/√N:

|s⟩ → (1/√N) Σ_{x≠ω} |x⟩ − (1/√N)|ω⟩

Medido directamente en este punto, el estado marcado sigue teniendo probabilidad (1/√N)² = 1/N — ¡nada ha cambiado en términos de probabilidad! La inversión de fase es invisible en una sola medición. Solo a través de interferencia con el operador de difusión (siguiente lección) se vuelve visible.

¿Cómo se construye el oráculo en la práctica?

En un circuito cuántico real, el oráculo depende del problema específico:

  • Búsqueda en base de datos: el oráculo verifica si x coincide con el elemento buscado (comparación bit a bit)
  • Satisfacibilidad (SAT): el oráculo evalúa una fórmula booleana en x y devuelve 1 si la satisface
  • Optimización: el oráculo verifica si una solución candidata cumple una cota mínima de calidad

La complejidad del oráculo depende del problema. Lo importante es que la ventaja cuadrática de Grover se mantiene independientemente de cómo esté construido, siempre que sea una puerta unitaria válida.

📦

El oráculo como abstracción: El modelo del oráculo es una abstracción teórica que permite analizar algoritmos independientemente del hardware. En la práctica, implementar el oráculo para problemas reales (como SAT con millones de variables) puede requerir muchos qubits auxiliares y circuitos profundos. El coste del oráculo es parte del análisis completo del algoritmo.

04

El operador de difusión de Grover

⏱ 24 min🟡 Intermedio✨ Gratis

El oráculo marca la solución con una fase negativa, pero eso no es suficiente para encontrarla. El segundo componente del algoritmo — el operador de difusión de Grover (también llamado "inversión sobre la media") — convierte esa diferencia de fase en una amplificación real de la probabilidad. Es el motor que hace que Grover funcione.

La inversión sobre la media

El operador de difusión realiza una operación geométrica elegante: refleja todas las amplitudes respecto a su valor medio. Si la media es μ, cada amplitud aₓ se transforma en 2μ − aₓ.

Veamos por qué esto amplifica la solución. Después del oráculo:

  • N−1 estados tienen amplitud +1/√N ≈ μ (un poco por encima de la media)
  • El estado marcado |ω⟩ tiene amplitud −1/√N (muy por debajo de la media)

La media μ es ligeramente menor que 1/√N. Al reflejar respecto a μ:

  • Los estados no marcados: su amplitud queda ligeramente reducida
  • El estado marcado: su amplitud grande y negativa se refleja a un valor grande y positivo
amplitud nueva = 2μ − amplitud vieja
Circuito del operador de difusión

El operador de difusión se implementa como: H⊗ⁿ · (2|0⟩⟨0| − I) · H⊗ⁿ. En circuito: (1) aplica H a todos los qubits, (2) aplica una puerta de fase que invierte todos los estados excepto |0⟩ (implementada con puertas X, una Toffoli de n-qubits y X de vuelta), (3) aplica H de nuevo. El coste es O(n) puertas con complejidad O(n²) si se descompone en puertas de 1 y 2 qubits.

La iteración de Grover: oráculo + difusión

Una iteración de Grover = oráculo + difusión. Geométricamente, cada iteración rota el estado cuántico en un ángulo de 2θ hacia el estado marcado |ω⟩, donde θ = arcsin(1/√N) ≈ 1/√N para N grande.

Para que el estado apunte a |ω⟩, necesitamos rotar un total de π/2 − θ ≈ π/2 radianes. Si cada iteración rota 2θ, el número de iteraciones necesario es:

k ≈ (π/4) · √N    iteraciones óptimas

Para N = 1.000.000: k ≈ (π/4)·1000 ≈ 785 iteraciones. Exactamente O(√N).

¿Qué pasa si iteramos demasiado?

Si pasamos del número óptimo de iteraciones, el estado "pasa de largo" y la probabilidad del estado marcado empieza a decrecer. Si seguimos iterando indefinidamente, la probabilidad oscila — sube y baja periódicamente. Esto significa que en la práctica hay que calcular el número correcto de iteraciones de antemano (necesitas saber N, el tamaño de la base de datos).

📐

Intuición geométrica completa: El estado cuántico es un vector en un espacio 2D. El oráculo lo refleja respecto al eje |ω⊥⟩. El operador de difusión lo refleja respecto al eje |s⟩. Dos reflexiones consecutivas = una rotación. La composición de las dos reflexiones rota el vector exactamente 2θ hacia |ω⟩ en cada iteración. Tras ~π√N/4 iteraciones, el vector apunta directamente a |ω⟩.

El algoritmo completo y su análisis

⏱ 22 min🟡 Intermedio✨ Gratis

Con todas las piezas en su lugar, ensamblamos el algoritmo de Grover completo y analizamos su complejidad, probabilidad de éxito y recursos necesarios.

El algoritmo de Grover paso a paso

  1. Inicializar: Prepara el estado |0⟩ⁿ en n qubits
  2. Superposición: Aplica H⊗ⁿ para crear la superposición uniforme |s⟩
  3. Repetir k ≈ (π/4)√N veces:
    • Aplica el oráculo U_f (invierte la fase del estado marcado)
    • Aplica el operador de difusión D = H⊗ⁿ(2|0⟩⟨0|−I)H⊗ⁿ
  4. Medir: Mide los n qubits en la base computacional
  5. Verificar: Comprueba clásicamente si el resultado es la solución
Probabilidad de éxito ≈ sin²((2k+1)θ) con θ = arcsin(1/√N)

Análisis de recursos

  • Qubits: n + (qubits del oráculo). El registro de búsqueda necesita n = log₂(N) qubits
  • Consultas al oráculo: O(√N) — la ventaja cuadrática
  • Puertas adicionales: O(n) por iteración para el operador de difusión
  • Puertas totales: O(n · √N) = O(√N · log N)
  • Probabilidad de éxito: ≥ 1 − 1/N para k = ⌊(π/4)√N⌋ iteraciones
Múltiples soluciones

Si hay M soluciones entre N elementos, el número óptimo de iteraciones se reduce a k ≈ (π/4)√(N/M). Con más soluciones, se necesitan menos iteraciones. Si M = N/4 (el 25% son soluciones), solo se necesita 1 iteración. Si M > N/2, el algoritmo de Grover puede "fallar" y hay que ajustar el planteamiento.

Grover con M soluciones desconocidas

En la práctica muchas veces no sabemos cuántas soluciones hay (M es desconocido). La solución es el algoritmo de Grover cuántico con búsqueda exponencial: empieza con 1 iteración, si no funciona dobla el número de iteraciones, y así sucesivamente. El número esperado de consultas sigue siendo O(√(N/M)) incluso sin conocer M de antemano.

📊

Ejemplo numérico: N = 1.000.000 elementos, 1 solución. k_óptimo ≈ 785. Después de exactamente 785 iteraciones, la probabilidad de éxito es >99.9%. Ejecutar el circuito una sola vez da el resultado correcto con casi total certeza. No hay que repetir el experimento muchas veces.

Variantes y extensiones de Grover

⏱ 20 min🟡 Intermedio✨ Gratis

El algoritmo de Grover original es el punto de partida, pero a lo largo de los años se han desarrollado variantes poderosas que amplían enormemente su aplicabilidad: Grover con amplificación de amplitud, búsqueda cuántica para optimización, y el algoritmo de Durr-Høyer para encontrar el mínimo de una función.

Amplificación de amplitud cuántica (QAA)

La amplificación de amplitud cuántica (Quantum Amplitude Amplification, QAA) es la generalización de Grover propuesta por Brassard, Høyer, Mosca y Tapp en 2002. En lugar de buscar un elemento específico, amplifica cualquier subespacio "bueno" de un estado cuántico.

Formalmente, si tienes un algoritmo cuántico A que produce el estado correcto con probabilidad p, la QAA da el resultado en solo O(1/√p) aplicaciones de A y A⁻¹, en lugar de O(1/p) repeticiones clásicas. Siempre que p > 0, hay ventaja cuadrática.

Repeticiones clásicas: O(1/p)     QAA: O(1/√p)

Grover para optimización: el algoritmo de Dürr-Høyer

El algoritmo de Dürr y Høyer (1996) usa Grover para encontrar el mínimo (o máximo) de una función f sobre un dominio de N elementos con solo O(√N) consultas:

  1. Elige un umbral inicial y aleatorio t
  2. Define el oráculo: marca todos los x con f(x) < f(t)
  3. Aplica Grover para encontrar un x' con f(x') < f(t)
  4. Actualiza t = x' y repite
  5. Tras O(√N) iteraciones totales, t converge al mínimo global

Esto da una ventaja cuadrática para optimización combinatoria sobre espacios no estructurados.

Grover adaptativo con número desconocido de soluciones

Cuando no sabemos cuántas soluciones M hay, la estrategia óptima es la búsqueda cuántica exponencial: comienza con j=1 iteraciones, aplica Grover y mide. Si falla, duplica j (j → cj con c = 6/5 en la versión óptima). El número esperado de consultas es O(√(N/M)) aunque M sea desconocido, lo que hace al algoritmo completamente práctico.

Grover en hardware NISQ actual

El algoritmo de Grover es uno de los pocos algoritmos cuánticos que se puede demostrar en hardware actual para casos pequeños. En IBM Quantum, puedes ejecutar Grover sobre n = 3 o 4 qubits (N = 8 o 16 elementos) con solo unas pocas iteraciones. Los resultados son visibles aunque el hardware tenga ruido, porque pocos pasos = poco error acumulado.

Para casos prácticos útiles (N > 2⁵⁰), se necesitaría hardware tolerante a fallos con muchos más qubits y fidelidades más altas que las actuales.

🔬

Grover en el laboratorio: El primer experimento de Grover fue realizado por Chuang, Gershenfeld y Kubinec en 1998 usando RMN (resonancia magnética nuclear) con 2 qubits, buscando en una base de datos de 4 elementos. Desde entonces se ha demostrado en fotónica, trampas de iones, qubits superconductores y puntos cuánticos.

Aplicaciones reales del algoritmo

⏱ 18 min🟡 Intermedio✨ Gratis🏁 Final

La ventaja cuadrática de Grover, aunque no es exponencial como la de Shor, tiene aplicaciones concretas en múltiples áreas. En muchos problemas prácticos, pasar de O(N) a O(√N) consultas puede ser la diferencia entre un problema computacionalmente intratable y uno que se puede resolver en tiempo razonable.

Criptografía: búsqueda de preimagen en hash

Las funciones hash (SHA-256, SHA-3) son usadas en firmas digitales, blockchains y contraseñas. Encontrar una preimagen (una entrada que produce un hash dado) requiere explorar el espacio de entradas de tamaño N = 2ⁿ. Grover lo hace en O(2^(n/2)) en lugar de O(2ⁿ), efectivamente reduciendo la seguridad de n bits a n/2 bits. Por eso SHA-256 sigue siendo seguro (se necesitan 2¹²⁸ operaciones cuánticas) pero SHA-1 (160 bits → 80 bits efectivos) se desaconseja.

Satisfacibilidad (SAT) y optimización combinatoria

El problema SAT (¿existe una asignación de variables booleanas que satisfaga una fórmula?) es NP-completo. Grover da una ventaja cuadrática incluso para SAT: en lugar de explorar las 2ⁿ asignaciones posibles, solo se necesitan O(2^(n/2)) consultas al oráculo que verifica si una asignación satisface la fórmula.

Lo mismo aplica a otros problemas NP como el viajante de comercio (TSP), coloración de grafos o empaquetamiento de mochila. La ventaja cuadrática no hace estos problemas polinomiales, pero puede dar speedups prácticos significativos.

Grover no hace NP ⊆ BQP

Aunque Grover da ventaja cuadrática en NP, no prueba que todos los problemas NP sean resolubles eficientemente por ordenadores cuánticos. Incluso con Grover, SAT sigue siendo O(2^(n/2)) — exponencial. La pregunta de si NP ⊆ BQP (problemas NP resolubles en tiempo polinomial cuántico) sigue abierta, y la mayoría de expertos creen que la respuesta es no.

Inteligencia artificial y machine learning

Varias subrrutinas de machine learning pueden beneficiarse de Grover: búsqueda del vecino más cercano, encontrar el máximo de una función de coste, y aceleración de la búsqueda en árboles de decisión. El algoritmo de Grover forma parte del "toolbox" de speedups cuánticos para ML junto con la QFT y la estimación de fase.

Bases de datos no estructuradas en la práctica

En bases de datos reales (SQL, NoSQL) las consultas son eficientes porque hay índices y estructura. La búsqueda no estructurada pura es menos común. Sin embargo, hay escenarios donde Grover aplica directamente: búsqueda en espacio de configuraciones de un problema de optimización, generación de números pseudoaleatorios cuánticos, y verificación de soluciones en problemas de decisión.

🚀

Hacia el futuro: Grover es uno de los algoritmos más implementados en IBM Quantum y otros procesadores cuánticos. Aunque el hardware actual solo puede ejecutarlo para N pequeño, sirve como banco de pruebas fundamental para medir la calidad del hardware. Cuando lleguen los ordenadores cuánticos tolerantes a fallos, Grover se convertirá en una subrutina estándar en muchos algoritmos cuánticos prácticos.

¿Qué efecto tiene Grover sobre la seguridad de SHA-256 (una función hash de 256 bits)?
✓ ¡Correcto! Grover reduce la seguridad de SHA-256 a 128 bits efectivos (√2²⁵⁶ = 2¹²⁸). 2¹²⁸ operaciones cuánticas es completamente inviable con cualquier tecnología imaginable. SHA-256 es seguro en la era cuántica.
